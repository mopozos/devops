# devops

Теоретическое задание:
1. нужно обеспечить высокую доступность сервиса, как бы ее
реализовал?
Ответ: Высокая доступность достигается за счет отказоустойчивости, резервирования и балансировки нагрузки.
Необходимо использовать мониторинг(и мониторинг на мониторинг, тк это тоже точка отказа :) ), балансировщики нагрузки, минимум 2-3 сервера с одинаковым контентом, и по необходимости мастер-сейв БД, если сервер хранит данные

3. тебе надо создать и настроить 5 таких серверов, как бы это
реализовал?
Ответ: Использовал бы инструменты автоматизации, чтобы не делать всё вручную(docker + k8s, ansible, terraform)
